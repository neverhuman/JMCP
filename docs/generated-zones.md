# Generated Zones

Generated files are governed by `agent/generated-zones.toml`. Treat that manifest as the source of truth before editing or regenerating contract outputs.

## Rules

- Do not hand-edit generated artifacts outside their declared zone.
- A generated file must name a reproducible local command and source path.
- Review source schema or generator diffs before reviewing generated output diffs.
- If a generated output changes, include the generator command and proof lane in the handoff or commit message.
- Human-authored docs stay human-authored unless `agent/generated-zones.toml` explicitly declares an exception.

## Current Declared Zone

`contracts/events/jcp-envelope.schema.json` is generated by `just contract-drift` from `schemas/jcp/1.0.0/jcp.schema.json`. Any change to that source schema must include the generated contract diff and the contract-drift receipt.

## Review Checklist

1. Confirm the output path appears in `agent/generated-zones.toml`.
2. Run the declared generator command from the repository root.
3. Run the mapped proof lane in `agent/test-map.json`.
4. Verify no generated output changed without a source or generator change.
5. Keep generated receipts under `target/jankurai/` unless the manifest says otherwise.
